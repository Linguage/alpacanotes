终于把博客自动同步到 Telegram 频道的功能彻底搞定了。

回顾一下这次折腾的过程，其实核心逻辑不难，细节全是坑：

  1. 基础链路：找 BotFather 申请机器人，把机器人拉进频道管理员，拿 Token 和 Chat ID。这步最简单。
  2. 自动化脚本：写了个 Node.js 脚本，配合 GitHub Actions。每次代码 push 到 main 分支时，用 git diff 算出新增的 Markdown 文件，解析出标题和链接发给 Bot。
  3. Instant View (即时预览)：这是最花时间的部分。
    • Telegram 的 IV 规则比较严格，比如 markdown 默认把图片包在 <p> 标签里，IV 直接报错，最后用 @split_parent 规则把它们拆开才解决。
    • 链接适配：发现电脑网页版 Telegram 打不开 t.me/iv 的链接。最后采用了“隐形链接”方案——在消息头塞一个指向 IV 的零宽空格链接用来触发预览卡片，明面上的“Read More”还是指向原始网页。这样既有卡片又能保证点击跳转正常。

现在流程终于跑通了：写文章 -> Push -> 自动触发 Action -> 频道秒推 (带 Instant View)。

舒服了，继续搬砖。
