#2:41 pm/telegram, 碎碎念

Feb 22, 2026

克莱伯在单届冬奥会中豪取六金的壮举，促使我又一次了解了一下奥运历史上的单届"多金王"。

纵观历届奥运会，单届狂揽6金以上的运动员屈指可数：除了2008年北京奥运会的菲尔普斯（8金）和1972年慕尼黑奥运会的马克·斯皮茨（7金）之外，与克莱伯同样单届拿下6金的，还有一位常常被忽略的传奇——1992年代表独联体出战的体操运动员维塔利·谢尔博。

谢尔博的成绩极具压迫感：在体操男子项目总共仅有8枚金牌的情况下，他一人独揽6金，甚至创下了单日狂揽4金的神迹，堪称前无古人。

然而，稍微深入了解他的故事后，却让人为他倍感惋惜。

谢尔博的职业生涯充满了时代带来的坎坷。他仅参加过两届奥运会：

1992年巴塞罗那奥运会：他代表由前苏联加盟共和国组成的松散代表团"独联体"参赛，留下了单届6金的神话。

1996年亚特兰大奥运会：苏联彻底解体后，他代表刚刚独立、经济十分困顿的白俄罗斯出战。

不仅缺乏大国的强力支持，谢尔博的备战还遭到了家庭变故的沉重打击。

1995年，他的妻子身患重病，谢尔博为此一度中断训练，竞技状态受到严重影响。即便如此，在1996年火线复出后，面对中国名将李小双和俄罗斯体操王子涅莫夫等顶尖高手的夹击，他在仅有白俄罗斯微薄资源支持的劣势下，依然拼下了4枚铜牌。这足以证明他拥有多么恐怖的体操天赋。

令人遗憾的是，尽管谢尔博在1992年达成了至今无人接近的历史级成就，他在体操史上的存在感却非常低。他不如苏联时期的柳金声名显赫，也不及后来的涅莫夫或中国的李小双家喻户晓。

这种"低存在感"，很大程度上是因为他生不逢时：

国家解体的遗憾：如果苏联晚解体几年，凭借其强大的国家宣传机器，谢尔博的壮举必将被大书特书；如果他能生在俄罗斯或乌克兰这样的大国，也能获得更高的曝光度和训练资源。

无人背书的荣誉：他的巅峰6金是代表一个已经不存在的临时组织（独联体）获得的，这导致后来没有哪个国家会主动将他作为民族体育英雄来大力宣传。

错失的封神契机：妻子重病阻碍了他的巅峰延续，如果1996年他能再夺几枚金牌，其历史地位必将无可撼动。

谢尔博的"单届六金"的辉煌最终只能相对沉寂地停留在奥运历史的档案中。

#8:54 pm/telegram, 碎碎念

Jan 24, 2026

终于把博客自动同步到 Telegram 频道的功能彻底搞定了。

回顾一下这次折腾的过程，其实核心逻辑不难，细节全是坑：

1. 基础链路：Bot + Actions

  • 申请机器人：找 BotFather 申请机器人，拉进频道当管理员，拿 Token 和 Chat ID。这步最简单。
  • 自动化脚本：写了个 Node.js 脚本，配合 GitHub Actions。每次代码 push 到 main 分支时，用 git diff 算出新增的 Markdown 文件，解析出标题和链接发给 Bot。

2. 问题一：文件重命名检测失效

  • 现象：修改了文件名（如 v1.md -> v2.md）后，推送没有触发通知。
  • 原因：Git 将此操作识别为 重命名 (Rename)，而脚本最初只检测 新增 (Add)。
  • 解决：将 git diff 的过滤器从 A 升级为 ACR，同时覆盖 Add、Copy、Rename 三种情况。

3. 问题二：Instant View 规则报错

  • 现象：IV 模板测试时，提示 <img> is not supported in <p>。
  • 原因：Markdown 渲染器默认将图片包裹在 <p> 标签里，但 IV 规范不允许。
  • 解决：在 IV 规则中加入 @split_parent: //p/img，强制将图片与父级段落拆开。

4. 问题三：Web 端链接无法打开

  • 现象：App 端点击 t.me/iv?... 链接能秒开，但 Web 端点击后无反应或报错。
  • 原因：Telegram Web 对 IV 链接的支持极差。
  • 尝试1（隐形链接）：在消息开头插入零宽空格 [\u200B](IV链接)，正文保留原始链接。这能在 App 端触发卡片，但依然不够优雅。
  • 尝试2（官方方案）：升级到 Telegram Bot API 的 link_preview_options。在 API 调用中显式指定 url: "t.me/iv?..." 用于生成预览卡片，而正文链接保持原始网址。这是目前最稳健的方案。

5. 问题四：通知没有预览卡片（时序问题）

  • 现象：消息能发送，但无论 App 还是 Web 都没有预览卡片。
  • 原因：通知步骤在部署步骤之前执行。当 Telegram 服务器收到链接去抓取时，新文章还未上线，返回 404，导致预览生成失败。
  • 解决：调整 GitHub Actions 工作流顺序，将通知步骤移到部署之后，并增加 10 秒延迟等待 CDN 生效。

最终流程

写文章 -> Push -> 自动触发 Action -> 先部署 -> 等待 10s -> 再通知 -> 频道秒推 (带 Instant View)。

舒服了，继续搬砖。

(Update: 优化了通知样式，现在 Note 会自动生成标题、摘要和标签了)

最近做了一些不一样的——把被闲置许久的网站进行了大改，增加和完善了许多功能。这个网站最早是用来分享内容，但在建站开发的过程中，我也会不断的思考许多交互的逻辑，把自己作为第一个严肃的用户对其反复拷打（事实上，除了我本人，也没有几个人看，感兴趣可以点击 linguista.cn ，你应该不会很失望）。当前AI在前端编程的能力非常恐怖，所以我并没有非常费神，而是频繁地获得惊喜，不断地为我提供正反馈。这种体验比最近形势惨淡的推特涨粉事业好了太多。

我不是专业开发者，因此我会在所有场合中都对真正专业的人士保持欣赏与尊重，这是让我能够从他们身上学到好东西的前提。我非常喜欢的一点就是，中推本身就云集了许多热衷于做产品的人，也有许多名副其实的大神级人物。通过一年多的观察，毫无疑问，技术区在中推中属于流量黑洞，但这并不重要，这个社区的朋友们是吸引我参与进来并不断地去表达的动力来源。

我希望自己是一个builder，不管build啥。我相信，这也将会成为越来越多人给自己添加的标签。没有什么事情能比在AI时代构建自己的创意更鼓舞人心。我在2024年年初终于“打通了向GitHub push垃圾的链路”之后，就不断地提交一些东西。一开始是一些笔记文档，后来开始clone或fork一些有趣的项目，然后自己修修改改，改得不成样之后放弃掉...

在很长时间里，我距离做出拿得出手的东西都相距甚远，但反复折腾本身就乐趣无穷。到后来，我慢慢地发现自己与专业的程序员交往不再有多少障碍。因为对各种产品有了更多内部视角的探索，也逐渐具备了一点能力去对一些事物发表个人看法。

但是，很遗憾，我生活在一个对代码世界极度无知的环境中，我周围的人大多对代码本身就充满恐惧。我需要有一个出口，让自己的热情能够持续下去。我逛过许多地方，最后发现X算是很难得的平台了。

当然，这里显然也不那么好，大家也都有自己的目的或志向，一天到晚熙熙攘攘，甚至算得上鱼龙混杂，偶尔“群魔乱舞”，不过几经困扰之后已经麻木。我很庆幸的是，做技术的人总会有一种保持纯粹的自觉性，我赖以生存的基本盘并没有在过去一年的算法迭代与风气演变中变质，反而有越来越多的新朋友加入。

我们生活在现实中，也生活在幻觉中。做一件事情，往往需要一些感召，这种感召源自于一个存在于脑海之中散发光芒、饱含热情的存在，源源不断地为我们提供战胜恐惧毅然前行的动力。我一度以为自己过于理想主义，应当向现实看齐。我也尝试过，但很不幸，那个现实对我而言过于无趣，我感受到的只有冰冷与残酷，没有激情的驱动与乐趣的诱惑，很快就如同行尸走肉。

当失去过一次之后，就会无比珍惜那些能够让自己热爱的事物。我还处于一个对自我的重建过程之中，不断搜集属于自己的能量碎片。编程、写作、与AI聊天、生成各种图片、向GitHub上提交垃圾...当然还有与有趣的推友们保持交流，这些都是这些年带给我能量的事情。

继续写作，继续构建，继续热爱！

Dan Koe最近爆火的长文。

他警告大家，工业时代的“专精”已成陷阱，我们正迎来属于博学者的“第二次文艺复兴”。

在这种趋势中，拥有多种兴趣非弱点，而是构建独特视角的终极优势。

我在昨天读过另一篇文章，也认为“通才的时代”要来了。可见这种观点不算标新立异。这是很多人能感知到的，正在默默地形成的一个共识。

了解Dan Koe的读者应该也会发现，这篇文章其实也没有什么很新颖的观点，而更像是对他一以贯之理念的一个汇总与延伸。

关于“通才”话题，未来几年肯定会被越来越多讨论的。通才并不是“样样通，样样松”，而是能够快速地从多个领域里及其思想与实践中的影响，从而可以用更高视角寻求对问题的理解与解决。

但通才不算是AI造就的。渴望博学是人的本能。在文艺复兴的年代，那些前沿的创作者，比如达芬奇与米开朗琪罗，他们都是通才。只是，在工业化时代，他们被不断细分的专业所割裂，不断地被边缘化。

在当前的经济环境与科技冲击下，过于细腻的专业分工逐渐展露其荒谬之处。而AI的突破，更是将掩埋通才的浮尘拂去。所以，Dan Koe认为这是“第二次文艺复兴”，当然也是“通才的复兴”。

那么，如何抓住这个机会呢？Dan Koe并不建议去追求那些本不属于自己的东西。与其看别人如何，不如利用好自己本就拥有的资源，技能、兴趣、激情，将自己品牌化，将自己的作品延伸到市场之中。

当然，我并不认为“通才”与“专才”有什么对立，人才在探索与解决问题的过程中，微观与宏观视角都很有价值，不同的问题的解决路径通常也需要两者结合，通一点或专一点，不过是技能光谱上自然的分布状态，而不是一种身份标签。

之所以有“通与专”这种区分，也是因为过去的社会组织者的一厢情愿。他们认为对人进行如此分割会让社会更为高效，但时代一者在变，过去合理的划分在当下已不合时宜。我认同Dan Koe的看法，我们要破除一些既有的社会观念带来的思维定势。

技术的进步会解放人们对某项工作的理解力和想象力。相反，技术的长期停滞反而会让我们误以为某项工作本该如此...

试想一下，八十年代玩初代马里奥画质的人是否能想象到现在3A大作的体验？

后来游戏引擎的发展可以让80年代的画质几乎不需要费劲就能轻松做出来，但这是否说明游戏画面的设计师就此完蛋了呢？

以为游戏画质本该如此而拒绝创新的人确实完蛋了，但是宫本茂没有止步于此; 在那个电子游戏萌芽的年代，很多年轻人也涌进来，贡献自己的创意，逐渐形成了当前的游戏生态。我们再回望当年，单从画质上看，现在的3A大作似乎已经成了另一类事物。

游戏行业的发展表明，通常随着技术的进步，对游戏的画质、美术、交互的要求总是更高、更细，对应的从业者也越来越多。

前端与游戏画质是比较接近的东西。人对视觉、交互体验方面的追求是无止境的，体验的提高会把人们的品味养刁，进一步提高对该行业的要求，相应地也就有了更多的工作需求。通常有了新技术之后只是开启了一个新世界。

我们无法根据现有的行业状态以及从业经验来认定未来会是什么样子。现在我们总是以技术停滞时代的思维惯性来看待正在迅速变革的行业。

"前端已死"就是这样一个例子。当然，我们刻板印象中的前端肯定要死了，但是服务于交互体验的前端会永远存在，而且正在进入全新的时代。

或许过几年再回头看，我们目前认为先进炫酷的前端设计就像习惯了3A大作之后的人们回看80年代的史前2D马赛克画质的游戏一样。

与失业问题相关的是对劳动力的需求。那么游戏产业对劳动力的需求是否在下降呢？

我看到一个数字，过去十多年游戏玩家的数量其实是在显著增长的（我看到一个数据，是从16年的十几亿增加到现在的30多亿）。

"我度过了完美的一生“

1951年4月29日，路德维希·维特根斯坦永远地闭上了眼睛。

他本可以成为历史上最不差钱的学者，但却放弃了所有财富，过着清苦的生活；

他是二十世纪最重要的哲学家，却仅以哲学为业，而不愿靠其谋生；

他人生的前半段开创了逻辑哲学，成为一代宗师，却在后半生致力于从相反的视角出发，创立了另一个门派——语言哲学。

听过刀郎的《罗刹海市》的应该有印象，他的父亲是奥地利的钢铁大亨，死后给他留下了丰厚遗产，足以让他成为欧洲最有钱的人之一。但他放弃了这份遗产，他认为金钱让人堕落。他一分钱都没有要，全部送给了自己的亲戚，理由是他认为这些人已经足够堕落，已无可救药。

彼时是一战刚结束，维特根斯坦在前线经历了战争，也一度沦为俘虏。他在战争期间陆陆续续地写下了他的思考，并结合之前的研究，在1920年前后整理出了著名的《逻辑哲学论》。

该书篇幅不大，但却是哲学史上最伟大的著作之一。只是这部作品给大约过了十年，才给维特根斯坦带来了一点点实在的回报。

这十年，几乎整个1920年代，维特根斯坦并没有继续他的哲学研究。他花了多年时间在奥地利乡村小学当老师，还编辑了一部非常成功的词典。后来，他为姐姐家设计了一栋小楼，也被认为非常经典的建筑。后来几经辗转，现在是维特根斯坦纪念馆。

维特根斯坦虽然已从哲学界隐退，但他的逻辑哲学论却一直是被学界研究与讨论的焦点。而当他终于再度出山之时，已经颇有一番“王者归来”的气势。彼时是大名鼎鼎的凯恩斯去火车站迎接他，而见面时凯恩斯的第一句话就是：

"上帝来了！"
